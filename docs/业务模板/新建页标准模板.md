# 新建页标准模板

> 适用于常见后台新建页：页面按业务分组展示表单，底部居中放置取消、提交按钮。

## 使用场景

- 列表页点击“新建”“新增”“发起申请”进入独立新建页面。
- 页面字段较多，需要按业务模块分块，例如基本信息、附件信息、审批信息。
- 表单需要支持三列布局、两控件联动、整行备注、上传文件、上传图片。
- 提交前需要统一校验，校验通过后调用业务接口。

## 组件组合

- 分组容器：`DetailSection`。
- 表单组件：`Form + useForm`。
- 基础按钮：`BaseButton`。
- 表单控件：优先使用 Form schema 内置组件；复杂联动项使用 `formItemProps.slots.default`。
- 路由跳转：列表页进入新建页，新建成功或取消后返回列表页。

## 布局约定

- 普通字段：三列布局，`span: 8`。
- 两控件联动字段：根据业务宽度选择 `span: 8` 或 `span: 16`；控件内部必须 `width: 100%` 撑满当前列。
- 备注、上传、长文本：整行布局，`span: 24`。
- 多个业务模块时，每个模块使用一个 `DetailSection`。
- 底部操作按钮固定放在页面底部区域居中展示，不放进 `DetailSection` 内容里。

## 路由配置

新建页通常不展示在菜单中，但需要允许从列表页跳转。隐藏路由必须配置 `canTo: true`，动态路由过滤场景建议配置 `followRoute`。

```ts
{
  path: 'create',
  name: 'DemoCreate',
  component: () => import('@/views/demo/Create.vue'),
  meta: {
    title: '新建',
    hidden: true,
    canTo: true,
    followRoute: '/demo/index',
    activeMenu: '/demo/index'
  }
}
```

如果项目开启服务端动态路由，后端或 mock 返回的路由列表也必须包含新建页路由，否则刷新或重新登录后仍可能 404。

## 标准骨架

```vue
<script setup lang="tsx">
import { reactive } from 'vue'
import { useRouter } from 'vue-router'
import { ElInput, ElOption, ElSelect, ElUpload } from 'element-plus'
import { DetailSection } from '@/components/DetailSection'
import { Form, type FormSchema } from '@/components/Form'
import { useForm } from '@/hooks/web/useForm'
import { useTagsView } from '@/hooks/web/useTagsView'

interface CreateFormValues {
  name?: string
  status?: string
  city?: string
  address?: string
  location?: string
  remark?: string
  uploadFile?: string
  uploadImage?: string
}

const router = useRouter()
const { closeCurrent } = useTagsView()

const statusOptions = [
  { label: '启用', value: 'enabled' },
  { label: '禁用', value: 'disabled' }
]

const cityOptions = [
  { label: '北京', value: 'beijing' },
  { label: '上海', value: 'shanghai' }
]

const singleColProps = {
  span: 8
}

const fullColProps = {
  span: 24
}

const basicFormSchema = reactive<FormSchema[]>([
  {
    field: 'name',
    label: '项目名称',
    component: 'Input',
    colProps: singleColProps,
    formItemProps: {
      rules: [{ required: true, message: '请输入项目名称', trigger: 'blur' }]
    },
    componentProps: {
      placeholder: '请输入项目名称'
    }
  },
  {
    field: 'status',
    label: '状态',
    component: 'Select',
    colProps: singleColProps,
    formItemProps: {
      rules: [{ required: true, message: '请选择状态', trigger: 'change' }]
    },
    componentProps: {
      placeholder: '请选择状态',
      options: statusOptions
    }
  },
  {
    field: 'location',
    label: '地点选择',
    component: 'Input',
    colProps: singleColProps,
    formItemProps: {
      rules: [{ required: true, message: '请选择城市并输入详细地址', trigger: 'change' }],
      slots: {
        default: (formModel: CreateFormValues) => {
          const syncLocationValue = () => {
            formModel.location =
              formModel.city && formModel.address ? `${formModel.city}/${formModel.address}` : ''
          }

          return (
            <div class="form-composite-field">
              <ElSelect
                modelValue={formModel.city}
                clearable
                placeholder="请选择城市"
                {...{
                  'onUpdate:modelValue': (value: string) => {
                    formModel.city = value
                    syncLocationValue()
                  }
                }}
                onChange={syncLocationValue}
              >
                {cityOptions.map((item) => (
                  <ElOption key={item.value} label={item.label} value={item.value} />
                ))}
              </ElSelect>
              <ElInput
                modelValue={formModel.address}
                clearable
                placeholder="请输入详细地址"
                {...{
                  'onUpdate:modelValue': (value: string) => {
                    formModel.address = value
                    syncLocationValue()
                  }
                }}
                onChange={syncLocationValue}
              />
            </div>
          )
        }
      }
    }
  },
  {
    field: 'remark',
    label: '备注',
    component: 'Input',
    colProps: fullColProps,
    componentProps: {
      type: 'textarea',
      rows: 3,
      placeholder: '请输入备注'
    }
  },
  {
    field: 'uploadFile',
    label: '上传文件',
    component: 'Input',
    colProps: fullColProps,
    formItemProps: {
      slots: {
        default: () => (
          <ElUpload action="#" autoUpload={false} limit={1}>
            {{
              trigger: () => (
                <button class="upload-file-trigger" type="button">
                  选择文件
                </button>
              ),
              tip: () => <div class="el-upload__tip">支持上传业务附件</div>
            }}
          </ElUpload>
        )
      }
    }
  },
  {
    field: 'uploadImage',
    label: '上传图片',
    component: 'Input',
    colProps: fullColProps,
    formItemProps: {
      slots: {
        default: () => (
          <ElUpload action="#" autoUpload={false} limit={1} listType="picture-card">
            {{
              default: () => <span class="text-24px leading-none">+</span>,
              tip: () => <div class="el-upload__tip">支持上传图片</div>
            }}
          </ElUpload>
        )
      }
    }
  }
])

const { formRegister, formMethods } = useForm()

const closePage = () => {
  closeCurrent(undefined, () => {
    router.push('/demo/index')
  })
}

const submitPage = async () => {
  const formExpose = await formMethods.getElFormExpose()
  const isValid = await formExpose?.validate().catch(() => false)

  if (!isValid) return

  const formValues = (await formMethods.getFormData()) as CreateFormValues

  // TODO: 调用新建接口，接口请求必须走 src/axios 封装。
  void formValues
  closePage()
}
</script>

<template>
  <div class="create-page">
    <DetailSection title="基本信息">
      <div class="detail-field--full">
        <Form :schema="basicFormSchema" label-width="90px" :is-col="true" @register="formRegister" />
      </div>
    </DetailSection>

    <div class="create-page__footer">
      <BaseButton @click="closePage">取消</BaseButton>
      <BaseButton type="primary" @click="submitPage">提交</BaseButton>
    </div>
  </div>
</template>

<style lang="scss" scoped>
.create-page {
  min-height: calc(100vh - var(--top-tool-height) - var(--tags-view-height) - 24px);

  &__footer {
    display: flex;
    padding: 12px 0;
    justify-content: center;
    gap: 12px;
  }
}

:deep(.form-composite-field) {
  display: flex;
  width: 100%;
  gap: 12px;

  .el-select,
  .el-input {
    flex: 1;
  }
}

:deep(.upload-file-trigger) {
  display: inline-flex;
  height: 34px;
  padding: 0 12px;
  appearance: none;
  color: var(--el-color-primary);
  font-size: 14px;
  font-weight: 500;
  cursor: pointer;
  background: #f5f7fb;
  border: none;
  border-radius: 4px;
  align-items: center;

  &:hover {
    background: #eef4ff;
  }
}
</style>
```

## 开发要点

1. 新建页优先使用 `Form + useForm`，不要在页面里手写大量重复 `el-form-item`。
2. 表单值必须定义类型，字段名与 schema 的 `field` 保持一致。
3. 普通输入、选择、日期等字段优先使用 schema 内置组件。
4. 两个控件组成一个业务字段时，使用聚合字段承接校验，例如 `location`。
5. 聚合字段内的子字段变化后，需要同步聚合字段值，否则必填校验无法准确触发。
6. 备注、说明、上传类字段默认占整行。
7. 提交前必须通过 `getElFormExpose().validate()` 做统一校验。
8. 取消或提交成功后，使用 `useTagsView().closeCurrent` 关闭当前标签页，再跳回列表页。
9. 新建接口请求必须放在 `src/api/`，并通过 `src/axios` 封装调用。
10. 新建页路由隐藏菜单时，必须配置 `hidden: true` 和 `canTo: true`。
11. 服务端动态路由模式下，后端路由或 mock 路由也需要同步增加新建页。
