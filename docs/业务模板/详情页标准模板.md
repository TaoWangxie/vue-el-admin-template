# 详情页标准模板

> 适用于常见后台详情页：按业务分组展示只读字段，通常由列表页点击“详情”进入。

## 使用场景

- 列表页需要跳转到详情页查看完整业务信息。
- 详情页以只读信息为主，不承担复杂编辑。
- 页面需要按模块分组，例如基本信息、处理结果、审批信息、附件信息。
- 详情页需要页内切换，例如车辆信息、时间线、附件、变更日志。

## 组件组合

- 分组容器：`DetailSection`。
- 页内切换：`DetailTabs`。
- 路由参数：`useRoute` 获取详情 `id`。
- 样式体系：SCSS、CSS 变量、Element Plus 变量。
- 菜单高亮：详情页路由通过 `activeMenu` 指回列表页。

## DetailSection

`DetailSection` 统一维护详情分组标题、标题竖线、内容栅格和字段基础样式，业务页面只写字段内容。

- `title`：分组标题。
- 默认插槽：分组内容。
- `title-top`：标题行上方的扩展内容，例如提示条、摘要信息、自定义工具区。
- `title-extra`：紧跟标题后的扩展内容，例如状态标签、提示。
- `header-extra`：标题行最右侧扩展内容，例如按钮、操作入口。

## 路由配置

详情页通常不展示在菜单中，但需要允许跳转。隐藏路由必须同时配置 `canTo: true`，前端动态路由过滤场景建议配置 `followRoute`。

```ts
{
  path: 'detail/:id?',
  name: 'DemoDetail',
  component: () => import('@/views/demo/Detail.vue'),
  meta: {
    title: '处置明细详情',
    hidden: true,
    canTo: true,
    followRoute: '/demo/index',
    activeMenu: '/demo/index'
  }
}
```

如果项目开启服务端动态路由，后端或 mock 返回的路由列表也必须包含详情页路由，否则刷新或重新登录后仍可能 404。

## 标准骨架

```vue
<script setup lang="ts">
import { computed, ref } from 'vue'
import { useRoute } from 'vue-router'
import { DetailSection } from '@/components/DetailSection'
import { DetailTabs, type DetailTabOption } from '@/components/DetailTabs'

const route = useRoute()

const detailId = computed(() => route.params.id as string | undefined)

const activeTab = ref<string>('vehicle')

const detailTabs: DetailTabOption[] = [
  { label: '车辆信息', value: 'vehicle' },
  { label: '车务时间线', value: 'timeline' },
  { label: '关联附件', value: 'attachment' },
  { label: '变更日志', value: 'log' }
]

const activeTabTitle = computed(() =>
  activeTab.value === 'vehicle'
    ? '车务信息'
    : detailTabs.find((tab) => tab.value === activeTab.value)?.label || ''
)
</script>

<template>
  <div class="detail-page">
    <DetailSection :title="activeTabTitle">
      <template #title-top>
        <DetailTabs v-model="activeTab" :options="detailTabs" />
      </template>
      <template #title-extra>
        <span class="ml-8px text-12px text-[var(--el-color-primary)]">审核中</span>
      </template>
      <template #header-extra>
        <BaseButton type="primary">操作</BaseButton>
      </template>

      <div class="detail-field">
        <span class="detail-field__label">车牌号：</span>
        <span class="detail-field__value"></span>
      </div>
      <div class="detail-field">
        <span class="detail-field__label">车架号：</span>
        <span class="detail-field__value">{{ detailId || '' }}</span>
      </div>
      <div class="detail-field">
        <span class="detail-field__label">配置款：</span>
        <span class="detail-field__value"></span>
      </div>
      <div class="detail-field">
```

## 列表页跳转

```ts
import { useRouter } from 'vue-router'

const router = useRouter()

const openDetailPage = (row: ListItem) => {
  router.push(`/demo/detail/${row.id}`)
}
```

## 页内切换建议

页内 tab 建议放在本模板中作为详情页可选结构，而不是单独拆一个“详情页切换模板”。这样对后续 AI 开发更友好：生成详情页时只需要读取一个模板，就能同时拿到路由配置、详情分组、页内切换和样式约定。

当 tab 内部承载复杂内容时，再补充专项模板，例如“详情页附件模板”“详情页时间线模板”。复杂内容单独拆，页面骨架留在本模板中。

## 开发要点

1. 详情接口返回值必须定义类型，避免长期使用隐式结构。
2. 详情分组使用 `DetailSection` 逐块写出，不使用 `v-for` 循环，便于按设计稿逐块调整。
3. 字段值由接口返回时，建议先定义详情接口，再直接绑定到对应字段节点。
4. 超过一行宽度的字段直接添加 `detail-field--full`，例如说明、备注、附件。
5. 不在业务页面重复编写 `detail-section`、`detail-field` 样式；样式由通用组件统一维护。
6. 详情页路由隐藏菜单时，必须配置 `hidden: true` 和 `canTo: true`。
7. 列表页进入详情后，`activeMenu` 指向列表页路径，保证菜单选中状态不丢。
8. 服务端动态路由模式下，后端路由或 mock 路由也需要同步增加详情页。
9. 页内 tab 仅用于详情页内部信息切换，不要替代一级菜单或 TagsView。
