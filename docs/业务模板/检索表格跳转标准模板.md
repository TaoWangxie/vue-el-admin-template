# 检索表格跳转标准模板

> 适用于常见后台列表页：上方检索，中间操作按钮，下方表格，新增/编辑跳转到独立表单页面。

## 使用场景

- 列表页需要查询、重置、分页、刷新。
- 表格需要操作列，例如详情、编辑、删除。
- 新增和编辑字段较多，适合用独立页面承载。
- 新增/编辑页面需要独立路由、独立 TagsView 标签和更完整的表单布局。

## 组件组合

- 检索区：`Search + FormSchema`
- 表格区：`Table + useTable`
- 页面容器：`ContentWrap`
- 路由跳转：`useRouter`
- 新增/编辑表单页：参考 `docs/业务模板/新建页标准模板.md`

## 标准骨架

```vue
<script setup lang="tsx">
import { reactive, ref } from 'vue'
import { useRouter } from 'vue-router'
import { ElButton, ElMessage, ElMessageBox, ElTag } from 'element-plus'
import { ContentWrap } from '@/components/ContentWrap'
import { Search } from '@/components/Search'
import { Table, type TableColumn } from '@/components/Table'
import { useTable } from '@/hooks/web/useTable'
import type { FormSchema } from '@/components/Form'

type Status = 'enabled' | 'disabled'

interface ListItem {
  id: number
  name: string
  status: Status
  createTime: string
}

interface QueryParams {
  keyword?: string
  status?: Status
}

const router = useRouter()

const statusOptions = [
  { label: '启用', value: 'enabled' },
  { label: '禁用', value: 'disabled' }
]

const queryParams = ref<QueryParams>({})

const searchSchema = reactive<FormSchema[]>([
  {
    field: 'keyword',
    label: '关键词',
    component: 'Input',
    componentProps: { placeholder: '请输入关键词' }
  },
  {
    field: 'status',
    label: '状态',
    component: 'Select',
    componentProps: {
      placeholder: '全部状态',
      options: statusOptions
    }
  }
])

const columns = reactive<TableColumn[]>([
  { field: 'index', label: '序号', type: 'index', width: 70 },
  { field: 'name', label: '名称', minWidth: 160 },
  {
    field: 'status',
    label: '状态',
    width: 100,
    slots: {
      default: ({ row }) => (
        <ElTag type={row.status === 'enabled' ? 'success' : 'danger'}>
          {row.status === 'enabled' ? '启用' : '禁用'}
        </ElTag>
      )
    }
  },
  { field: 'createTime', label: '创建时间', width: 140 },
  {
    field: 'action',
    label: '操作',
    width: 160,
    fixed: 'right',
    slots: {
      default: ({ row }) => (
        <div class="flex items-center gap-8px">
          <ElButton type="primary" link onClick={() => openDetailPage(row as ListItem)}>
            详情
          </ElButton>
          <ElButton type="primary" link onClick={() => openCreatePage(row as ListItem)}>
            编辑
          </ElButton>
          <ElButton type="danger" link onClick={() => deleteRecord(row as ListItem)}>
            删除
          </ElButton>
        </div>
      )
    }
  }
])

const { tableRegister, tableState, tableMethods } = useTable({
  immediate: true,
  fetchDataApi: async () => {
    // 替换为真实接口：getListApi({ ...queryParams.value, page, pageSize })
    return { list: [], total: 0 }
  }
})

const reloadTable = () => tableMethods.getList()

const searchList = (params: QueryParams) => {
  queryParams.value = params
  tableState.currentPage.value = 1
  reloadTable()
}

const resetList = (params: QueryParams) => {
  queryParams.value = params
  tableState.currentPage.value = 1
  reloadTable()
}

const openCreatePage = (row?: ListItem) => {
  router.push({
    path: '/demo/create',
    query: row ? { id: row.id } : undefined
  })
}

const openDetailPage = (row: ListItem) => {
  router.push(`/demo/detail/${row.id}`)
}

const deleteRecord = async (row: ListItem) => {
  try {
    await ElMessageBox.confirm(`确认删除“${row.name}”吗？`, '删除确认', {
      confirmButtonText: '确定',
      cancelButtonText: '取消',
      type: 'warning'
    })
  } catch {
    return
  }

  // await deleteApi(row.id)
  ElMessage.success('删除成功')
  reloadTable()
}
</script>

<template>
  <ContentWrap class="mb-12px">
    <Search
      :schema="searchSchema"
      :is-col="true"
      :column-count="4"
      label-width="80px"
      @search="searchList"
      @reset="resetList"
    />
  </ContentWrap>

  <ContentWrap>
    <div class="mb-12px flex items-center justify-between">
      <BaseButton type="primary" @click="openCreatePage()">新增</BaseButton>
    </div>

    <Table
      v-model:current-page="tableState.currentPage.value"
      v-model:page-size="tableState.pageSize.value"
      row-key="id"
      :columns="columns"
      :data="tableState.dataList.value"
      :loading="tableState.loading.value"
      :pagination="{ total: tableState.total.value }"
      show-action
      stripe
      @register="tableRegister"
      @refresh="reloadTable"
    />
  </ContentWrap>
</template>
```

## 路由配置

新增/编辑页通常不展示在菜单中，但需要允许跳转。隐藏路由必须配置 `canTo: true`，动态路由过滤场景建议配置 `followRoute`。

```ts
{
  path: 'create',
  name: 'DemoCreate',
  component: () => import('@/views/demo/Create.vue'),
  meta: {
    title: '新建',
    hidden: true,
    canTo: true,
    followRoute: '/demo/index',
    activeMenu: '/demo/index'
  }
}
```

编辑时可以通过 query 或 params 把业务主键传给表单页。当前模板使用 query：

```ts
router.push({
  path: '/demo/create',
  query: { id: row.id }
})
```

## 开发要点

1. 检索条件变化后，先把 `currentPage` 重置为 `1`，再刷新表格。
2. 搜索项列数优先用 `Search` 的 `columnCount` 或 `responsiveColProps` 控制。
3. 新增和编辑统一跳转到表单页，列表页不再维护表单状态。
4. 编辑跳转时必须携带业务主键，表单页根据主键判断新增或编辑并回显数据。
5. 表单页取消或提交成功后，需要关闭当前 TagsView 标签并返回列表页。
6. 删除操作必须先弹确认提示，用户确认后再调用删除接口并刷新表格。
7. 真实接口应拆到 `src/api/业务模块/`，请求参数和返回类型必须显式声明。
